## 编程

- 本机开发环境 Windows10 + pwsh(Powershell 7)
- 默认禁止使用 Bash 语法， 除非确定shell处在 Linux 环境
- 执行多行 Python 禁止使用 Bash heredoc；改用 PowerShell here-string | python -，或写入临时 .py 文件后运行。

- 尽量使用 guard 语句,遵循提前返回原则,避免使用嵌套过深的语句。
- 合理拆分文件，防止出现很长的代码文件
- 不允许擅自删除修改用户的注释，除非注释明确错误。
- 迁移/重命名优先一次性修改完成，禁止任何兼容层、别名等代码。
- 确保函数只负责一项职责。

#### 编程原则
原则01: `如无必要 勿增实体`
  - 复用现有的抽象和接口，不要新增实现除非它确实解决了明确的职责问题
  - 在新增实现时只需满足当前业务需求，**不要提前设计**目前没需求的功能

原则02: `DRY -- Don't Repeat Yourself`
- 尽量复用已有的抽象，约定，逻辑，函数。
- 提取公共实现/类型，尽量让多处代码只依赖一处公共定义。
- 避免重复逻辑；当重复逻辑确实存在且当前需要时，提取共享的辅助函数。

#### 测试
- 除非我明确要求 否则不要自动执行验证测试
- 测试脚本，优先进行失败断言，保证失败立即暴露
- 测试脚本，日志只保留关键成功/失败结果，不去层层输出运行细节

#### 计划任务
- 对于简单的任务，可以直接进行。
- 对于复杂的任务，首先制定一个简短的计划。
- 如果存在重大歧义阻碍执行，在进行重大更改之前进行咨询。


## MCP工具
- 用 Context7 去查各类文档

### 使用 dc(desktop-commander MCP)
- 调用 dc 的 start_process 时:
  - 固定使用的shell为 pwsh.exe 禁止使用 powershell.exe
  - 注意启动shell后默认的工作目录是 C:\Windows\System32 , 所以需要使用 Set-Location 切到合适的路径后再执行命令
- 读取普通文本文件时优先使用 read_multiple_files
- 禁用 list_processes 命令。必须改用 start_process 调用 pwsh Get-Process
